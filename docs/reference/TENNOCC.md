
# Forge — Complete Application Blueprint

> **Forge** is an offline-first Warframe build planner, DPS calculator, and theory-crafting tool built as an Electron desktop application with React. It parses the full `@wfcd/items` data set to provide real-time statistical analysis of Warframe, weapon, and companion builds with no server dependency.

## 5. THE STAT PROCESSOR ENGINE (src/engine/stat-processor/ — 1120 lines)

### 5.1 Overview

The processor takes a `BuildCore` + `ItemResolver` and synchronously produces a `CalculatedStats` object. All calculation is deterministic, single-pass, with no async.

### 5.2 Modifier System (src/modifier.ts)

Every item produces **Modifier** objects:

```typescript
interface Modifier {
  stat: string;             // e.g. 'base_damage', 'crit_chance', 'strength'
  category: 'FLAT' | 'MULTIPLIER';
  value: number;            // FLAT: raw number; MULTIPLIER: decimal (1.65 = +165%)
  stackingGroup: string;    // Items with same group are processed together
  source: string;           // Human-readable origin (for tooltips)
  priority: number;         // Execution order (0 = base, 1 = mods, 2 = arcanes)
  condition?: ModifierCondition;  // Conditional activation
}
```

**Resolution rule**: All modifiers within the same `stat::stackingGroup` are:
1. FLAT values summed together
2. MULTIPLIER values summed, then applied as `(1 + sum)` × flat_total
3. Final = FLAT_sum × (1 + MULTIPLIER_sum)

### 5.3 Processing Pipeline

```
BuildCore → resolveWarframePassive
          → resolveMods (normalMods)
          → resolveMods (aura)
          → resolveMods (exilus)
          → resolveArcanes
          → resolveShards
          → buffs
          → sortMods (by priority)
          → applyConditions (filter/scale conditional mods)
          → bucketify (group by stat::stackingGroup)
          → combineElementals (slot-order elemental pairing)
          → adjustModifiers (aiming remapping)
          → compute ability stats (strength, duration, range, efficiency)
          → compute warframe stats (health, shields, armor, energy, sprint, EHP)
          → compute weapons (per weapon: damage, multishot, crit, status, DPS, DOT)
          → compute companion
          → compute enemy damage (if enemy state present)
          → collect set bonuses
```

### 5.4 Elemental Combination Logic

Implements Warframe's slot-order elemental pairing:

```
SINGLE_ELEMENTALS: heat, cold, electric, toxin

COMBINATIONS:
  heat + cold = blast        cold + heat = blast
  heat + electric = radiation  electric + heat = radiation
  heat + toxin = gas          toxin + heat = gas
  cold + electric = magnetic  electric + cold = magnetic
  cold + toxin = viral        toxin + cold = viral
  electric + toxin = corrosive  toxin + electric = corrosive

Rule: Same-element mods stack additively (merged before pairing).
Pairs: 1st+2nd → combined, 3rd+4th → combined (slot order).
```

### 5.5 Weapon Damage Calculation

```
1. Base Damage:
   totalBase = (flat_base_damage) × (1 + Σ base_dmg_mult + CO_bonus × activeStatuses)

2. Faction Multiplier:
   factionMult = 1 + Σ faction_damage_mods (if targetFaction matches)

3. Damage Types:
   For physical (impact/puncture/slash):
     damage = base_value × finalBaseDmgMult × (1 + physical_type_mult) × factionMult
   For elemental mods:
     damage = totalBase × elemental_type_mult × factionMult
   For innate elemental:
     damage = base_value × finalBaseDmgMult × factionMult

4. Multishot:
   multishot = flat × multiplier

5. Critical:
   critChance = base_crit × crit_mult
   critTiers: yellow = min(chance, 1), orange = max(0, min(chance-1, 1)), red = max(0, chance-2)
   critFactor = 1 + critChance × (critMultiplier - 1)

6. Fire Rate, Reload, Magazine (from weapon base + mods)

7. DPS:
   avgShotDamage = totalDamage × multishot × critFactor
   burstDps = avgShotDamage × fireRate
   sustainedDps = (avgShotDamage × magazine) / (magazine/fireRate + reloadSpeed)

8. Status Probabilities (4x weight for physical types):
   weighted = damage_value × (4 for physical, 1 for elemental)
   probability = weighted / totalWeighted × 100

9. DoT (Damage over Time):
   slashBleed: 0.35 × baseDmg × factionMult² per tick, 7 ticks over 6s
   heatBurn: 0.5 × baseDmg × heatModMult × factionMult² per tick, 7 ticks over 6s
   toxin: 0.5 × baseDmg × factionMult² per tick, 11 ticks over 10s
   gas: 0.5 × baseDmg × factionMult² AoE per tick, 7 ticks over 6s
   electric: 0.5 × baseDmg × factionMult² AoE per tick, 3 ticks over 3s
   viral proc: doubles all DoT damage (viralUptime-based scaling)

10. Heavy Attack (melee only):
    heavyCritChance, heavyWindUp, heavyEfficiency, initialCombo, comboDuration, comboChance
```

### 5.6 Warframe Ability Stats

```
final = base × (1 + Σ mod_bonuses)
  strength, duration, range work as above
  efficiency capped at 1.75 (175%)

Helminth ability damage:
  scaledDamage = baseDamage × (1 + (scalingStat - 1) × scalingFactor)
```

### 5.7 Enemy Damage Resolution

When `BuildCore.enemy` is present:

```
1. Level scaling:
   scaledArmor = baseArmor × (1 + level)^0.75
   scaledHealth = baseHealth × (1 + level)^0.5
   scaledShields = baseShields × (1 + level)^0.6

2. Armor reduction:
   corrosive: removes 25% of remaining base armor per stack (compound)
   armorStripped: fractional reduction (0-1)
   heatProc: multiplicative 50% reduction

3. Damage type modifiers:
   vs health: (1 + healthTypeMod + armorTypeMod) × (1 - DR)
   vs shields: (1 + shieldTypeMod)
   Where DR = effectiveArmor / (effectiveArmor + 300)

4. TTK:
   shotsToBreakShield = ceil(scaledShields / perShotVsShield)
   shotsToKillHealth = ceil(scaledHealth / perShotVsHealth)
   burstTtk = totalShots / fireRate
   sustainedTtk includes reload cycles

5. Effective DPS:
   effectiveDps = (scaledShields + scaledHealth) / sustainedTtk
   multiTargetEffectiveDps = effectiveDps × multiTarget
```

### 5.8 Conditional Modifier System

Galvanized mods and conditional bonuses are handled via `ModifierCondition`:

- `galvanizedStacks`: Value × min(stackCount, maxStacks)
- `onHeadshot`: Boolean include; stackable variants multiply by stacks
- `onKill`, `onSlashProc`, `onAimGlide`, `onWallLatch`, `onSlide`, `onSpawn`: Boolean include

Without triggers → all conditional mods dropped.
With triggers → only matching mods included, scaled appropriately.

## 7. WFCD RESOLVER (src/data/wfcd-resolver.ts — 644 lines)

### 7.1 Purpose

Translates WFCD (warframe-items) JSON entries into the app's canonical `Modifier[]` format that the stat processor can consume.

### 7.2 Stat Mapping

The `STAT_MAP` is the largest single data structure in the app (~190 entries). Each WFCD stat string maps to a normalized modifier:

| WFCD stat string | Internal stat | Group | Category |
|---|---|---|---|
| "Damage" | base_damage | weapon_base_damage | MULTIPLIER |
| "Multishot" | multishot | weapon_multishot | MULTIPLIER |
| "Critical Chance" | crit_chance | weapon_crit | MULTIPLIER |
| "Critical Chance when Aiming" | aiming_crit_chance | weapon_crit | MULTIPLIER |
| "Heat" | elemental_heat | weapon_elemental | MULTIPLIER |
| "Ability Strength" | strength | ability | MULTIPLIER |
| "Health" | health | warframe_health | MULTIPLIER |
| "Armor" | armor | warframe_armor | MULTIPLIER |
| "Shield Recharge" | shield_recharge | warframe_shields | MULTIPLIER |

| "Melee Damage per Status Type affecting the target" | co_damage | weapon_co | MULTIPLIER |
| ...190 more entries | ... | ... | ... |

### 7.3 Stat Resolution Flow

```
WFCD stat string (e.g. "+165% Damage")
  → parseStatLine: regex extraction => { name: "Damage", value: 165, isPercent: true }
  → STAT_ALIAS resolution (e.g. "Power Strength" → "Ability Strength")
  → STAT_MAP lookup => { stat: "base_damage", category: "MULTIPLIER", stackingGroup: "weapon_base_damage", scaleByHundred: true }
  → scaleByHundred: 165 / 100 = 1.65
  → detectCondition: checks for "when Aiming", "for Xs", "Stacks up to Nx" etc.
  → Emit Modifier { stat: "base_damage", category: "MULTIPLIER", value: 1.65, ... }
```

### 7.4 Custom Riven Support

Rivens bypass the levelStats system entirely. When a mod has `rivenData` (positives[] + optional negative), the resolver directly maps each positive stat through the stat map, applying `scaleByHundred` for percentage-based stats.

### 7.5 Weapon Passive Resolution

Weapon base stats (totalDamage, multishot, critChance, critMultiplier, fireRate, procChance, reloadTime, magazine) are emitted as FLAT modifiers at priority 0, forming the baseline that mod MULTIPLIERs amplify.

Damage type arrays (`damagePerShot[]` indexed array or `damageTypes` record) are emitted as `damage_{type}` FLAT modifiers.

## 9. ENEMY SIMULATOR (src/enemy-simulator.ts)

### 9.2 Effective Armor


## 20. IMPORTANT MATH FORMULAS (SUMMARY)

### Damage
```
totalBase = flat_base × (1 + Σ baseMult + CO_bonus × statusTypes)
shotDamage = totalBase × (1 + elementalMult + physicalMult) × factionMult
avgShot = shotDamage × multishot × (1 + critChance × (critMult - 1))
burstDPS = avgShot × fireRate
sustainedDPS = (avgShot × mag) / (mag/fireRate + reloadTime)
```

### EHP
```
ehp = health × (1 + armor/300) + shields
```

### Enemy Armor
```
effectiveArmor = base × (1+level)^0.75 × (1-strip) × (1-0.25)^corr
DR = effectiveArmor / (effectiveArmor + 300)
```

### Status
```
weight = damage × (4 if physical, 1 if elemental)
proc% = typeWeight / totalWeight × 100
```

### DoT
```
Slash: 0.35 × base × faction², 7 ticks/6s
Heat:  0.50 × base × (1+heatMod) × faction², 7 ticks/6s
Toxin: 0.50 × base × faction², 11 ticks/10s
Gas:   0.50 × base × faction², 7 ticks/6s (AoE)
Elec:  0.50 × base × faction², 3 ticks/3s (AoE)
Viral multiplier: ×(1 + viralUptime) on all DoT
```

### Modifier Resolution
```
FLAT_sum = Σ flat values (same stat, same stacking group)
MULT_sum = 1 + Σ multiplier values
final = FLAT_sum × MULT_sum
```

| **P0** | **Sustained DPS bug** | `sustainedDps` calculation double-counts burst DPS because `shotsPerCycle = magazine` instead of scaling by magazine capacity. Currently `(avgShot * magazine) / (magazine/fireRate + reloadSpeed)` should account for magazine but may have off-by-one with partial reload. Verify: sustained DPS should be `(avgShot * magazine) / (magazine/fireRate + reloadSpeed)` with magazine being the actual capacity. |
| **P0** | **Reload time formula** | Current: `reload / max(reloadMult, 0.01)`. This uses `reload_speed_mult` but doesn't use `reload_speed` as the base. The flat reload_speed value is set from weapon's `reloadTime` but consumed differently. Verify: reload math matches Warframe: `effectiveReload = baseReload / (1 + reloadSpeedMods)`. |
